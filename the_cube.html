Introduction

This project was part of the Minor Internet of Things after I switched from Game Development to Technical Informatics. The minor had two separate assignments, each lasting 10 weeks. This page is dedicated to the individual project, "The Cube".

Process

This was my first time working with C++, which took some time to get used to. Python was not allowed, so I had to learn C++ from scratch. Several blueprints were available to choose from, but given my experience with making games, I chose the handheld device blueprint.

The Cube is a cubical handheld device with buttons on every face and a small LCD display on the front that gives players prompts to follow. The goal is to follow the prompts as quickly as possible. Each time a prompt is completed, the player scores a point and gets less time for the next prompt. The game has three modes:

  • Normal Mode: The standard mode where you follow prompts as fast as possible.
  • Easy Mode: You have double time for the prompts, but the score is not added to the database.
  • Colourblind Mode: Instead of prompts like "press the blue button", it says "press the left button".

Hardware

The Cube is made with an ESP32, which connects to the internet to store and access players' scores. The ESP32 is connected to the LCD display, the buttons, and the sensors that detect whether the player has completed the prompt correctly.
